Payment Required

The web reserved a status code for money in the nineties and never shipped it. AI agents are about to need it — badly.

By Akshat Gada — I build agentic payments at Polygon.

Open the HTTP spec and scroll past the status codes everyone knows. 200 OK. 404, the famous one. Keep going. You'll find 402 — Payment Required. It has been sitting in the spec since 1997, marked "reserved for future use."

The future took a while.

Here's the strange thing about the internet: we made it astonishingly good at moving information and left it helpless at moving money. A server in Tokyo answers your request in 80 milliseconds. Paying that server a tenth of a cent for the work takes a card number, a billing address, a 3% fee, a chargeback window, and a human clicking "Confirm." We built a global nervous system and bolted a 1994 cash register onto the side of it.

For thirty years that was fine, because the thing making the request was a person. People are slow. People have wallets. People can wait for a checkout page to load and squint at a CAPTCHA.

Agents are not people.

The unit of payment is about to get very small

An AI agent doesn't buy one thing and go home. It makes a request, reads the answer, makes ten more, hires another agent to do the part it's bad at, pays for an API call, rents a slice of compute, checks a price, tries again. Thousands of tiny actions, most worth a fraction of a cent, none worth interrupting a human to approve.

You cannot run that on credit cards. The fee is bigger than the purchase. There's no one at the keyboard to type the CVV. The entire model assumes a person at checkout — and the person is gone.

So you get two choices. Invent some elaborate new payment protocol and spend a decade dragging the world onto it — or use the one the web already reserved a status code for.

We went with the second one. Obviously.

x402, in one breath

It's almost boring how simple it is, which is exactly the point.

GET /price → 402 Payment Required ↳ pay 0.001 USDC · Polygon · 0x… pay, then retry GET /price → 200 OK { …the data }

The agent asks for something. The server says "that'll cost you," in a header, with machine-readable terms. The agent pays — stablecoins, onchain, no account, no login — and asks again. This time it gets a 200 and the thing it wanted. No checkout page. No human. No new app, no new browser. It's just HTTP doing what HTTP already does, plus the part we skipped in 1997.

We didn't reinvent the web. We finished it.

The boring part is the important part

Telling a server to "accept crypto" sounds easy until you try it. Now every API has to watch a chain, confirm a payment landed, handle the one that didn't, eat the gas, and not tip over when ten thousand agents hit it at once. Nobody is going to do all that to sell a tenth of a cent of data.

So I spent a good chunk of last year building the thing in the middle — the facilitator. It sits in front of the server, takes the payment, verifies it, settles it, and hands back a clean yes or no. The server stays a normal server. It never has to learn what a nonce is.

I wrote it in Rust, because it lives in the hot path of every single request, and "usually fast, occasionally on fire" is not a spec you ship. It's open source. It's running.

Then there's the part everyone forgets: incentives. You can have the cleanest protocol on earth and it dies on the shelf if adoption is annoying. So we wrote PIP-82 — a program that rebates up to a million dollars in gas to the facilitators carrying these payments. Make the rail cheaper than the friction it replaces and you don't have to convince anyone. They just switch.

Ideology doesn't ship. Incentives do.

The day it stopped being a demo

I remember the first time an agent paid for something with no human anywhere near it. It hit a 402, read the terms, signed, paid, retried, got its data, and went on to the next thing. A couple hundred milliseconds, start to finish. No confetti. No notification. Nothing.

Completely anticlimactic — which is how I knew it was real. Good infrastructure is boring. Electricity is boring. TCP is boring. You only notice it when it's missing, and pretty soon you won't be able to imagine it missing.

Where this goes

Picture a few million agents, each with a wallet, each able to pay any other agent or service per request, settling in stablecoins at machine speed, around the clock, while we sleep. Agents hiring agents. Markets that clear in milliseconds for amounts no human would bother with. An economy running underneath the one we can see.

That's not a metaphor, and it's not five years out. The status code has been waiting since the nineties. We finally gave it something to do.

402, at last.
